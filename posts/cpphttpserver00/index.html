从零开始的Http服务器学习00–项目搭建

本系列文章，想从零开始使用cpp实现一个http服务器，并在其中逐渐优化并添加各种技术，来达到学习的目的。

首先，说明一下开发环境，目前是在mac上进行开发和测试，随着内容的深入会迁移到Linux上去，还会在之后使用docker进行编译和部署。

项目说明

本项目的全部源码可在该页面查看。

创建项目

先是创建完毕的项目目录结构。

├── CMakeLists.txt
├── LICENSE
├── README.md
├── include
└── src
    └── main.cpp

本文是这个系列的第00篇文章主要是架构目录，简单介绍一些目标，所以这里面最开始main.cpp的内容会非常简单。

//src/main.cpp
#include<iostream>
using namespace std;
int main(){
    cout<<"project httpserver"<<endl;
    return 0;
}

准备好了源码，接下来就是对于项目的编译了。现在项目是单文件，所以可以直接使用g++ src/main.cpp -o httpd --std=c++17来进行编译，然而随着项目的扩大，源码文件会越来越多，此时就需要编译套件的出场了。 体现在项目中就是CMakeLists.txt文件。

cmake_minimum_required(VERSION 3.11)
project(httpserver-tutorial)
set(CMAKE_CXX_STANDARD 17)

add_executable(httpd src/main.cpp)

当CMakeLists.txt文件编辑完毕之后，我们就可以开始编译过程了。我这边使用vscode的cmake插件可以轻松完成编译和运行的过程，但是还是展示一下手动编译该如何做吧。

首先，在项目根目录创建build文件夹，我们接下来的操作都会在build文件夹下面进行操作。下面是完整的操作流程。

mkdir build
cd build
cmake ..
make
./httpd

分步骤来说就是利用cmake工具和配置好的CMakeLists.txt文件来生成对于的makefile文件，然后使用make命令进行编译即可，最后运行一下httpd即可。

最后的结果：

project httpserver

正常输出，我们的项目结构就搭建完毕了。下一篇，我们将开始对于套接字编程进行初步了解。
